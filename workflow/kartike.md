## **📅 30-Day Web Development Execution Plan**

---

## **PHASE 1 — Foundations & Learning (Days 1–7)**

  

### **🎯 Goal**

  

Reach _minimum working confidence_ in React + API-driven UI.

---

### **Days 1–2: Orientation & Setup**

  

**Tasks**

- Clone GitHub repo
    
- Create frontend branch
    
- Setup React project
    
- Install Tailwind or MUI
    
- Understand full system flow:

**Deliverables**

- React app runs locally
    
- Folder structure finalized
    
- README skeleton updated
    

---

### **Days 3–5: Focused Learning (No Overkill)**

  

**Learn ONLY the following:**

- React fundamentals:
    
    - Components
        
    - Props
        
    - State (useState)
        
    - Side effects (useEffect)
        
    
- File upload handling
    
- Axios POST requests
    
- Conditional rendering (loading / error / result)
    
- Basic responsive layout
    
- CSS positioning (for heatmap overlay)
    

  

**Target Outcome**

- You can upload an image and hit an API
    
- You can display dynamic results
    

---

### **Days 6–7: Mini Practice Build**

  

**Tasks**

- Build dummy UI:
    
    - Image upload
        
    - Submit button
        
    - Mock API response
        
    
- Display:
    
    - Label (Real / Forged)
        
    - Confidence score
        
    

  

**Deliverables**

- Fully working UI with mock data
    
- Clean component structure
    

---

## **PHASE 2 — Core Build (Days 8–15)**

  

### **🎯 Goal**

  

Deliver a **working end-to-end MVP**.

---

### **Days 8–9: Frontend–Backend Integration**

  

**Tasks**

- Connect React app to FastAPI backend
    
- Send image using multipart/form-data
    
- Handle:
    
    - Loading state
        
    - API errors
        
    - Invalid files
        
    

  

**Deliverables**

- Image upload → real prediction
    
- No UI freezing
    
- Clear error messages
    

---

### **Days 10–11: UX & Flow Improvements**

  

**Tasks**

- Improve upload UX:
    
    - Drag & drop (optional)
        
    - File preview
        
    
- Add prediction confidence visualization
    
- Prevent multiple submissions
    

  

**Deliverables**

- Smooth, intuitive flow
    
- No accidental misuse
    

---

### **Days 12–13: UI Polish**

  

**Tasks**

- Design result card:
    
    - Prediction label
        
    - Confidence %
        
    
- Improve spacing, typography, alignment
    
- Ensure mobile responsiveness
    

  

**Deliverables**

- UI looks professional
    
- Not “college-project-looking”
    

---

### **Days 14–15: MVP Freeze**

  

**Tasks**

- End-to-end testing
    
- Remove unused code
    
- Lock UI features
    

  

**Deliverable**

- Stable MVP (no feature additions after this)
    

---

## **PHASE 3 — Advanced Features & Quality (Days 16–23)**

  

### **🎯 Goal**

  

Turn UI into a **presentation weapon**.

---

### **Days 16–17: Heatmap Overlay (Explainability UI)**

  

**Tasks**

- Display original image
    
- Overlay Grad-CAM heatmap
    
- Add opacity slider
    
- Toggle heatmap on/off
    

  

**Deliverables**

- Clear explainable AI visualization
    
- Judges can “see” manipulation
    

---

### **Days 18–19: Metadata Display Panel**

  

**Tasks**

- Display EXIF metadata received from backend
    
- Highlight suspicious fields
    
- Keep layout readable (accordion or panel)
    

  

**Deliverables**

- Secondary analysis visible
    
- UI supports non-ML insights
    

---

### **Days 20–21: Evaluation Visualization**

  

**Tasks**

- Add model evaluation images to UI or slides:
    
    - Confusion matrix
        
    - ROC curve
        
    
- Optional: metrics summary cards
    

  

**Deliverables**

- Judges see real numbers, not claims
    

---

### **Days 22–23: Bonus Feature (Choose ONE)**

  

Pick **only one**:

- Batch image upload UI
    
- PDF report download button
    
- Demo mode (preloaded sample images)
    
- History of last predictions (local only)
    

  

**Deliverable**

- One standout UI feature
    

---

## **PHASE 4 — Testing, Polish & Presentation (Days 24–30)**

  

### **🎯 Goal**

  

Demo-proof the application.

---

### **Days 24–25: Edge Case Handling**

  

**Tasks**

- Handle:
    
    - Unsupported formats
        
    - Large images
        
    - Empty uploads
        
    
- Improve user feedback messages
    

---

### **Days 26–27: Documentation & Cleanup**

  

**Tasks**

- Write:
    
    - Frontend setup instructions
        
    - Folder structure explanation
        
    
- Add screenshots to README
    
- Clean GitHub commits
    

---

### **Days 28–29: Presentation Prep**

  

**Tasks**

- Prepare slides covering:
    
    - UI flow
        
    - Architecture diagram
        
    - Feature walkthrough
        
    
- Capture UI screenshots
    

---

### **Day 30: Final Review**

  

**Tasks**

- Live demo rehearsal
    
- Backup demo recording
    
- Final UI release tag
    

---

## **🚫 What NOT To Do**

- Do not rewrite UI mid-project
    
- Do not add animations for fun
    
- Do not introduce Redux or complex state tools
    
- Do not ignore error handling
    

---

## **✅ What Judges Notice Instantly**

- Clean layout
    
- Fast response
    
- Clear results
    
- Explainable visuals
    
- Confident demo flow
